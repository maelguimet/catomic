## Shipped Contract

- A Linux-first modeless terminal editor with Unicode-safe PieceTable editing,
  incremental line indexing, selection, search, undo/redo, atomic save, external
  change protection, multiple buffers, and editable paged large files.
- Linux saves refuse non-regular targets, hard-linked files, and files carrying
  xattrs or ACLs rather than silently replacing incompatible filesystem
  semantics. Replacement also fails unless mode, owner, and group match.
- Viewport-only light syntax, Markdown preview, line/whitespace indicators,
  bounded Plain completion, and explicitly gated Project discovery, linting,
  diagnostics, and cached path completion.
- Explicit, bounded, confirmed current-file and repository-brokered LLM flows;
  validated single-file output, read-only preview, drift refusal, and one-step
  undo. No client or network call exists before explicit confirmation, and API
  keys are refused for non-loopback plaintext HTTP endpoints.
- Typed TOML, language settings, keybinding overrides, trusted but guarded named
  commands, ordered lifecycle hooks, and preview-before-edit semantics.
- Restrained cat polish plus opt-in private `.catnap` recovery that never replaces
  source automatically.

## Pre-publication Secret Scan

- The official gitleaks 8.30.1 Linux archive matched its published SHA-256
  checksum.
- `gitleaks git --log-opts="--all"` scanned 481 commits / 2.27 MB across every
  then-present ref and found no leaks.
- A separate stdin scan covered 108.78 KB of issue and pull-request titles,
  bodies, issue comments, review comments, review bodies, and commit comments;
  it found no leaks.

The scans were completed while the repository was still private and before any
visibility change. They are evidence for this point in time, not a guarantee
that future commits or newly added issue content are secret-free.

## Deliberate Non-Goals

The completed v0.1 contract does not include a full LSP client, tree-sitter,
embedded scripting/plugin ABI, multi-file LLM apply, `:feralmeow`, split views,
non-UTF-8 filenames, non-Linux first-class support, or a heavy theming/widget
framework. These are explicit non-goals, not incomplete Phase 0–8 exit criteria.

## Result

Catomic v0.1 is implementation-complete against its roadmap and the first beta
candidate passes the complete source, manual, supply-chain, MSRV, package, and
release-build gate.
